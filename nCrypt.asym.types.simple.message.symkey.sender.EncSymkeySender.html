Class: EncSymkeySender

nCrypt.asym.types.simple.message.symkey.sender. EncSymkeySender

new EncSymkeySender(obj, skey, sym_alg [, sym_opts])

Messages are (potentially / often) encrypted for more than one receiver. This is why it makes sense to encrypt the symmetric key the message is encrypted with using the shared secrets, instead of encrypting the whole message (possibly) many times. (The message is usually longer than the symmetric key, which would result in unnecessarily long messages and long calculation time.)
An array of encrypted symmetric keys should be appended to the encrypted message, so a receiver can try to decrypt a symmetric key using a shared secret between sender and receiver.
This encrypted symmetric key objects should contain the actual encrypted symmetric key, the receiver's public key ID (so the receiver can find out which symmetric key in the array to decrypt without trial and error), the type of shared secret ('ecies' or 'dh'), and in case of ECIES, the tag required to restore the shared secret.
This class creates such an object from an instance of a shared secret class (nCrypt.asym.types.shared.dh.SecretDH or nCrypt.asym.types.shared.ecies.SecretECIES), a symmetric key and symmetric encryption options. Using the symmetric encryption options, the @symkey will be encrypted using the shared secret.
The serialized version (available as JSON string or parsed JSON) should be appended to an array of encrypted symkey objects and passed to the receiver along with the encrypted message.
Please note: If passing an instance of this class as the first argument, a clone will be returned.

Parameters:
Name Type Argument Description
obj string | nCrypt.asym.types.shared.dh.SecretDH | nCrypt.asym.types.shared.ecies.SecretECIES

The shared secret, i.e. an instance of one of the shared secret classes. A serialized instance (string) will work as well.
skey nCrypt.asym.types.basic.Secret | string

The symmetric key. This can either be a secret (easily created from a string), or a string. In case of a string, an instance of nCrypt.asym.types.basic.Secret will be created, using the string as a value and assuming a serialized instance of nCrypt.asym.types.basic.Secret as a source.
sym_alg string

Symmetric algorithm, for example 'aes', 'twofish' or 'serpent'. Needs to be supported in nCrypt.sym.
sym_opts object <optional>

Symmetric encryption options.
