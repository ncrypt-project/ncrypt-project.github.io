new Shared(symkey, sym_alg[, sym_opts])

Get a shared secret object to use for message encryption. This is derived from either an instance of nCrypt.asym.types.shared.dh.SecretDH or nCrypt.asym.types.shared.ecies.SecretECIES, and a symmetric key which will be used for message encryption.
The shared secret object will contain the encrypted symmetric key (the symmetric key will be encrypted with the actual shared secret as a key), an ID of the receiver's public keyset (so the receiver will know they should which symmetric key to decrypt if there are multiple receivers), and (in case of an ECIES derived shared secret) the ECIES tag.
Please note this class is NOT used to derive a shared secret. It rather takes a shared secret object (SecretDH or SecretECIES), and uses it to encrypt a symmetric key (usually used for message encryption, and decrypted by the receiver so they can decrypt the message). The result are the encrypted symmetric key and the information a receiver needs - secret type ('ecies' or 'dh'), possible ecies tag, and the receiver's public key ID so the owner of this keyset knows which encrypted symmetric key to decrypt.

getSerialized  function()

Get the serialized 'Shared' object. This can NOT be used to recreate a new instance of 'Shared', but instead is what you pass along with a message.
It contains the public key ID of the receiver (so they recognize this shared key data is for them to decrypt), the encrypted symmetric message key, and - in case of ECIES - the tag required to restore the shared secret. It also contains a 'type' property to check whether the shared secret to decrypt the symmetric key was derived using ECIES ('ecies') or DH ('dh').
