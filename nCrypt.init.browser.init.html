initialise(trustBrowserRNG, onUpdate, onDone)

Initialize nCrypt in browser, i.e. on a webpage. As this potentially includes collecting random data from random mouse- or touchmoves, advice users to move their mouse or fingers on touchpad as randomly as possible and give feedback on progress (see param @onUpdate).
If a browser RNG is found and trusted (don't rely on this), initialisation will be really fast, even to fast for a user to actually notice. They might see a small flickering on the screen when exchanging initialisation GUI by application GUI.
However, even if browser random data is trusted and available in all your clients, this proceeding is necessary, as even auto-collecting of random data takes a few milliseconds and only after that, nCrypt can be used normally.
When initialisation is done, @onDone is called. Only then you nCrypt is ready for use, so this function should contain code to load the actual application view / start application functions.
NEVER use nCrypt without initialising - symmetrical as well as asymmetrical encryption will be incredibly insecure without sufficient random data collected! In fact, security of nCrypt depends on the randomness of collected data.
A few words concerning whether to trust the browser random generator (if available) or not: Both works well, so it depends on both users and security needs. When trusting the browser RNG, manual random value generation will be just a fallback for users whose browser do not support generating cryptographically random values. Most browsers support random value generation, so this will be much more convenient for users. When users are really likely to use decent browsers, or security needs aren't THAT high, it should be fine to save users the cursor moving work.

Parameters

trustBrowserRNG

boolean

 

Whether to trust the browser random generator.

onUpdate

function()

 

This function is called with one parameter, the initialisation progress, i.e. the progress of nCrypt's random number collectors collecting random numbers for cryptographic use. The progress is passed to @onUpdate({number} prg}) as a percent value. As collecting random values can take quite a while if a browser RNG is not present or trusted, and requires users to move their mouse (or finger on touchpad) as randomly as possible, use this function to update users on initialisation progress.

onDone

function()

 

Called when the collectors are seeded, i.e. nCrypt is ready to go. Respond in GUI by proceeding to actual application.
